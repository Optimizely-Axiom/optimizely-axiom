# CSS Layers

All Axiom styles are declared inside an [@layer](https://developer.mozilla.org/en-US/docs/Web/CSS/@layer) rule called `optiaxiom` to allow controlling the order of precedence of styles.

```css
@layer optiaxiom {
  /* all Axiom styles */
}
```

We also declare a layer called `optiaxiom.base` where you can place your own styles so they come first in the cascade.

```css
@layer optiaxiom.base {
  /* these styles are placed before other Axiom styles */
}
```

## Usage

### Global styles

Global styles include reset/normalize rules and rules that target elements without any class modifiers. These global styles can conflict with styles coming from Axiom.

Place your global style rules inside the `optiaxiom.base` layer to ensure they are applied before any of Axiom's other styles.

```css {1,5}
@layer optiaxiom.base {
  a {
    /* your global styles */
  }
}
```

### Tailwind

Place your Tailwind base directive inside the `optiaxiom.base` layer to prevent conflicts with styles coming from Axiom.

#### Tailwind CSS 4.0

```css {3}
@layer theme, components, utilities;
@import "tailwindcss/theme.css" layer(theme);
@import "tailwindcss/preflight.css" layer(optiaxiom.base);
@import "tailwindcss/utilities.css" layer(utilities);
```

#### Tailwind CSS 3.0

```css {1-3}
@layer optiaxiom.base {
  @tailwind base;
}

@tailwind components;
@tailwind utilities;
```
